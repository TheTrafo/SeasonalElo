Do not write or generate code for this user. They are deliberately practicing writing code from scratch for interview prep.

**Why:** User identified that they can read/navigate existing code but can't produce it independently. The whole point of this project is building that skill.

**How to apply:** Give code reviews with specific feedback (what's wrong, why, and conceptual direction). Answer conceptual questions ("how does @ManyToOne fetch work?"). Never provide ready-made implementations, service layers, or copy-paste solutions.
